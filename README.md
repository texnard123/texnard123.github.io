# Сайт для бібліотеки — шаблон

Готовий каркас сайту, який можна віддати будь-якій бібліотеці: 3 інформаційні сторінки + сторінка новин, що підвантажується з Google Таблиці. Без сервера, без бекенду, без плати за хостинг.

## Структура
- `index.html` — головна (презентація ідеї / вітальна сторінка бібліотеки)
- `features.html` — можливості шаблону / послуги бібліотеки
- `how-it-works.html` — покроковий запуск / детальна інформація
- `news.html` + `news.js` — новини (тягнуться з Google Таблиці)
- `catalog.html` + `catalog.js` + `catalog.css` — електронний каталог (тягнеться з Google Таблиці, картка з двома режимами перегляду)
- `style.css` — спільна стильова система
- `images/` — сюди класти картинки для новин

## Як адаптувати під конкретну бібліотеку
1. Заміни назву й тексти в `<title>`, `.brand-title`, `.brand-sub` та в тексті сторінок на назву та опис своєї бібліотеки.
2. Створи власну Google Таблицю з колонками **Назва**, **Опис**, **Зображення** (назва файлу картинки, а не посилання на Google Drive).
3. Відкрий доступ до таблиці: **Поділитися → "Усі, хто має посилання" → Читач**.
4. У файлі `news.js` заміни `SHEET_ID` на ID своєї таблиці (це рядок символів у посиланні між `/d/` та `/edit`).
5. Картинки заливай у папку `images/` через "Add file → Upload files" на GitHub.

## Каталог видань
Сторінка `catalog.html` працює за тим самим принципом: підвантажує дані з окремої Google Таблиці (колонки ID, ISBN, Автор, Назва, Відомості, Відп., Видання, Місце, Видавець, Рік, Обсяг, УДК, Тематика, Ел. ресурс). Клік по картці відкриває модальне вікно з двома режимами перегляду — «Стисло» (основне для читача) і «Детально» (повний бібліографічний опис). Щоб підключити свою таблицю, заміни `SHEET_ID` у файлі `catalog.js`.

## Деплой на GitHub Pages
1. Створи репозиторій `<назва-бібліотеки>.github.io`, залий у корінь усі файли шаблону.
2. Settings → Pages → Source: гілка `main`, папка `/root`.
3. Сайт з'явиться за адресою `https://<назва-бібліотеки>.github.io/`.
4. За бажання підключи власний домен через Settings → Pages → Custom domain.

## Якщо новини на іншому аркуші таблиці
У `news.js` є змінна `SHEET_GID` — візьми gid з адресного рядка потрібного аркуша таблиці (число після `gid=`) і встав туди.
